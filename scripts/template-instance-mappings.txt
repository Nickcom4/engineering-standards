# Template-to-instance mappings for lint-template-compliance.sh
#
# Format (one mapping per line):
#   template-path | instance-glob | instance-type-label
#
# Lines beginning with # are comments. Blank lines are ignored.
#
# One template can have multiple mapping lines when it applies to
# multiple instance path patterns (e.g., fmea.md applies to both
# DFMEA-*.md and PFMEA-*.md).
#
# Instance-glob is relative to the repo root and follows shell glob
# syntax. Instance files matched by the glob must contain every
# non-optional ## section from the template, or declare the omission
# with an inline <!-- omit-section: Name --> comment.
#
# Adding a new mapping: append a line. No code change required.
# Adding a new template instance type: create the template (mark any
# optional sections with <!-- optional --> on the heading line),
# create the instance directory, add the mapping line.
